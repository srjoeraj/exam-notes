Using Essential File Management Tools.

The Filesystem Hierarchy Standard (FHS)

  • Directory usage on Linux is highly standardized.
  • Standard directories are defined in the FHS which is maintained by Linux Foundation.
  • The starting point of the filesystem is the root (/) directory.
  • Different devices may be integrated in the filesystem by using mounts.
  • man hier will show the man page for FHS.

/bin

  • contains binaries or user executable files which are available to all users.

/sbin

  • contains applications that only the superuser (hence the initial s) will need.

/boot

  • contains files required for starting your system.

/home

  • where you find user's home directories. Under this directory, there is another directory for each user, if that particular user has a home directory.

Note: root has it's home directory seprated from the rest of the users home directories and is /root

/dev

  • contains device files.

/etc

  • contains most if not all system-wide config files.

/lib

  • contains shared library fiels used by different applications.

/media

  • it is used for external storage and mostly is auto mounted.

/mnt

  • it like /media but not used often these days.

/tmp

  • contains temporary files, usually saved there by applications that are running. Non priv users may also store files here temporarily.

/proc

  • it is a virutal directory. It contains info about your computer hardware, such as cpu info, RAM memory, kernel etc. The files and directories are generated when your system starts, or on the fly, as your system is running and things change.

/sys

  • contains info about devices, drivers, and some kernel features.

/srv

  • contains data for servers.

/run

  • It is a temporary filesystem that runs in RAM.

/usr

  • contains many other sub directories binaries files, shared libraries and so on. On some distro like CentOS many commands are saved in /usr/bin and /usr/sbin instead of /bin and /sbin

/var

  • it typically contains variable length files such as logs which are files that register events that happen on the system.

Using links

Archiving Files and Working with Compressed files

  • The tar command.
  • Can be used compress, extract or list
  • tar -cvf my_archive.tar /folder1 /folder2 to create an archive
  • tar -tvf archive will show contents of an archive.

  • tar -xvf archive extracts contents of an archive.

  • To create archive not in pwd, use -C.

  • To add compression, use -z, -j, -J, before f.

  • A wide range of compression solutions is available for linux.

  • gzip (-z) is still the most common compression utility.
  • bzip2 (j) is an alternate utility.
  • zip is also available and has windows compatible syntax.
  • xz (-J) is showing up more often as well.

Using ROOT Privileges

Understanding the root user

  • In UNIX, a user account with UID 0 exits to perform adminstrative tasks.
  • On linux, this user by default is named root.

  • The root user operates in the kernel space and for that reason, has unlimited access to all parts fo the system.

  • While installing RHEL 9, the root user can be activated or not.

  • If the root user is not activated, the adminstrative user is used for admin tasks.
  • To activate the root user, you'll just have to set a password
  • From a security perspective, it may be a good idea not to have an active root user.
  • Additional measuers are possible to restrict root user access to a system.
  • Remote root login using secure shell (SSH) can be allowed or not

su command

  • Using su to switch to root shell is considered bad practice. use sudo -i instead.

Using sudo command

  • sudo is a more secure mechanism to perform adminstration tasks.
  • Behind sudo is the /etc/sudoers config file.
  • While editing /etc/sudoers through visudo very detailed adminstration privileges can be assigned.
  • To run an adminstration taks using sudo, use sudo command.
  • This will prompt for the current user password and run the command if this is allowed through /etc/sudoers
  • To open a root shell, sudo -i can be used.

Managing sudo configuration

  • sudo config is in /etc/sudoers.
  • should not be edited directly, use visudo, as it also does some checks on the sudoers file.
  • instead of editing /etc/sudoers, create and drop in files in /etc/sudoers.d

  • /etc/sudoers is installed from packages and may be overwritten, drop-in files never be overwritten.

  • In RHEL, users that are members of wheel group get full sudo access.

  • Accomplished by the following line %wheel ALL=(ALL) ALL in /etc/sudoers.
  • Use usermod -aG wheel user to add user to wheel group.
  • Do NOT enable the line %wheel ALL=(ALL) NOPASSWD: ALL
  • it provides full sudo access without using a passowrd!.

  • Use the following line, to increase the authentication token expiration:

    • Defaults timestamp_type=global, timestamp_timeout=60

  • Use drop-in files to provide admin access to specific tasks

  • lisa ALL=/sbin/useradd, /usr/bin/passwd
  • Consider using command arguments to make the commands more specific
  • %users ALL=/bin/mount /dev/sdb, /bin/umount, /dev/sdb
  • linda ALL=/usr/bin/passwd, ! /usr/bin/passwd root

Managing Users and Groups

  • A user is a security priniciple, user accounts are used to provide people or processes access to system resources.
  • Processes are using system accounts.
  • People are using regular user accounts.

Setting User Properties

  • User properties are managed in /etc/passwd
  • Name: the name of the account
  • Password: the users's password, may be disabled.
  • UID: user id.
  • GID: primary group id.
  • GECOS: additional non-mandatory infor about the user.
  • Home directory: the env where users create personal files.
  • Shell: the program that will be started after successful authentication.

  • usermod -U anna will unlock anna.

Managing Group Membership

  • Each user must be a member of at least one group.
  • Primary groups membership is managed through /etc/passwd
  • The user primary group becomes group-owner if a user creates a file.
  • Additional (secondary) groups can be defined as well
  • Secondary group membership is managed through /etc/groups
  • Temporarily set primary group membership using newgrp.
  • Use id to see which groups a user is a member of

Creating and Managing Groups

  • Use groupadd to add groups.
  • groupdel and groupmod can be used to delete and modify groups
  • Use lid -g groupname to list all users that are members of specific group.
[root@localhost student]# groupadd sales
[root@localhost student]# groupmod -U anna sales
[root@localhost student]# id anna

Securing Files With Permissions

Understanding Ownership

  • To determine permissions a user has, Linux uses ownership.
  • Every file has user-owner, group-owner, and the other entity that is also granted permissions (ugo).
  • Linux permissions are not additive, if you're the owner, permissions are applied and that's all.
  • Use ls -l to display current ownership and associated permissions.
  • Best Practice: Set ownership before modifying permissions.

Change File Ownerhsip

  • chown and chgrp commands
[root@localhost data]# chown lisa newfiles/
[root@localhost data]# ls -l
total 0
drwxr-xr-x. 2 lisa root 6 Oct 30 11:45 newfiles

[root@localhost data]# ls -l

Understanding Basic Permissions

Files Directory
READ (4) Open the file Allows to list the directory
WRITE (2) Modify file create / delete files
EXECUTE (1) run create sub directories etc

Special X

  • When x is applied recusively, it would make directories as well as files executable, this can be dangerous.
  • In recusive command, use X instead
  • Directories will be granted the execute permission.
  • Files will only get the execute permission if it is set already elsewhere on the file.

Managing Basic Permissions

  • Permission do not apply to the root user.
  • chmod can be used in absolute mode chmod 751 myfile or in relative mode chmod +x myscript.

The thing to remember, is creating and deleting file depends on the directory permission and not on permission on the file created.

Configuring Shared Group Directories

  • If members of the same group need to share files within a directory, some special permissions are required
  • The Set Group ID (SGID), permission ensures that all files in the shared group directory are group owned by the group owner of the directory.

  • The sticky bit permission ensures that only the user who is owner of the file, or the directory that contains the file, is allowed to delete the file.

  • chmod g+s mydir will apply SGID to the directory

  • chmod +t mydir will apply sticky bit to the directory.

Understand umask

  • The umask is a shell setting that subtracts the umask from the default permissions
  • Default is et in /etc/bashrc
  • Set user-specific overrides in ~/.bashrc
  • Default permission for file are 666
  • Default permissions for directory are 777

Managing Network Configuration

  • In IPv4 each node needs its own IP address, written in dotted deciaml notation (192.168.4.200/24).
  • Each IP address must be indicated with the subnet mask behind it
  • The default router or gateway specifies which server to forward packets to that have an external destination.
  • The DNS nameserver is the IP address of a server that helps to resolve to IP addresses and the other way around.
  • IPv4 is still the most common IP version, but IPv6 addresses can be used asd well.
  • IPv6 addresses are written in hexadecimal notation.
  • IPv4 and IPv6 can co-exist on the same network interface.

Exploring IPv6 networking

  • IPv6 was introduced in the 1990's to overcome the shortage of world-wide unique IPv4 addresses.
  • IPv6 is used extensively by ISP's to address the core internet infrastructure
  • End-users and companies mostly use IPv4 behind a NAT router.

  • Linux distros (including RHEL), offer dual stack IPv4 and IPv6

  • IPv6 addresses are 128-bit numbers, which are normally expressed as 8 colon-seprated groups of four hexadecimal numbers.

  • In these numbers, leading zero's are omitted.

  • 2001:db:7891:123:1010:6bbd:cbcb:210

  • Long strings of zeros can be replaced by one block of two colon characters

  • ::210

  • When combining an IPv6 address with a port number, enclose the IPv6 address in sq brackets

  • [2001:bad:210]:80

  • Typically, network providers hand out a /48 prefixed address which leaves 16 bits to the customer to address subnets

  • Different IPv6 address types are available

Purpose Example Description
localhost ::1/128 Local host address, used on the loopback interface
unspecified :: Used to refer to all IP addresses
default route ::/0 Used in default route specification
global unicast 2000::/3 IPv6 addresses currently being allocated
unique local fd00::/8 Addresses for internal use like 192.168.0.0
link local fe80::/10 Non-routable auto assigned for internal use.
multicast ff00::/8 Multicast addresses
  • Apart from manual allocation and DHCP IPv6 supports Stateless Address Autoconfiguration (SLAAC)
  • In SLAAC, a host sends a router solicitation to the ff02::2 multicast group to access all routers
  • A router answers that request sending all relevant information
  • The host adds an automatically generated host ID to the network prefix to obtain a unique address in this way
  • To enable RHEL 9 for SLAAC support, install the radvd package

Understanding NIC naming

  • IP address configuration needs to be connected to a specific network device.
  • Use ip link show to see current devices, and ip addr show to see their configuration

  • Every system has a lo device, which is for internal networking.

  • Classical naming is using device names like eth0, eth1 and so on

  • These device names don't reveal any info about physical device location.

  • BIOS naming is based on hardware properties to give more specific information in the device name.

  • em[1-N] for embedded NIC's
  • eno[nn] for embedded NIC's

  • pp for NIC's on the PCI bus

  • If the driver doesn't reveal network device properties, classic naming is used.

Defining Host name and Host name resolution

  • /etc/resolv.conf contains DNS client config
  • The order of host name resolution is determined through /etc/nsswitch.conf. as follows:

Analyzing Network config

  • DO NOT USE THE IPCONFIG COMMAND
  • ip addr or ip a to view network properties.
  • ip addr add dev ens33 10.0.0.10/24
  • ip link to show link properties
  • ip route to show route properties
  • ip route show

  • ip route add default via 10.0.0.1

  • Changes made by ip command are NOT PERSISTENT, use NetworkManager instead.

Understanding Network Manager

  • NetworkManager is the system service that manages network configuration.
  • Configuration is stored in files in /etc/NetworkManager/system-connections

  • Legacy files in /etc/sysconfig/network-scripts are still supported but deprecated.

  • Different applications are available to interface with NetworkManager

  • nmcli is a powerful command line utility.
  • nmtui offers a convenient text user interface.

  • GNOME offers graphical tools also.

  • In NetworkManager, devices are network interfaces

  • Connections are collected of configuration settings for a device, stored in the config file in /etc/NetworkManager/system-connections

  • Only one connection can be active for a device.

  • Permissions to modify settings in NetworkManager are applied through dbus.

  • Non priv users that are logged in on the console can change network settings
  • Non priv users that are logged in through ssh cannot
  • Use nmcli general permissions for an overview of current permissions that apply.

Network Troubleshooting

  1. Use the ping command
  2. ping -c 4 myserver sends 4 packets then stops

  3. ping6 2001:210 for ipv6

    • When using ping6 with link-local addresses, you must include NIC name in the command.
    • ping6 ff02::1%ens33

  4. ip route prints routing table andip -6 route shows IPv6 routing table.

  5. tracepath example.com show the entire networking path and tracepath6 example.com does the same for an IPv6 path (if exists).

  6. ss command to analyze stock statistics
  7. ss -tu
  8. ss -tuna
  9. ss -tunap most recommended, shows tcp, udp, all info, with port numbers.
